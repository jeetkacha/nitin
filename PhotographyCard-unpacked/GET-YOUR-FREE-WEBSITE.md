---

### **Step 2: Create Repository**

1. Go to: https://github.com/new
2. Fill in:
   - **Repository name:** `nitin-art-studio`
   - **Description:** "NITIN ART STUDIO Digital Business Card"
   - **Visibility:** ✅ Public
   - **DO NOT** check "Add README"
3. Click **"Create repository"**

---

### **Step 3: Push Your Code**

Open Terminal and run:

```bash
cd ~/Downloads/PhotographyCard

# Switch to main branch
git branch -M main

# Stage all files
git add .

# Commit
git commit -m "NITIN ART STUDIO business card"

# Add your repository (REPLACE YOUR-USERNAME!)
git remote add origin https://github.com/YOUR-USERNAME/nitin-art-studio.git

# Push to GitHub
git push -u origin main
```

**⚠️ IMPORTANT:** Replace `YOUR-USERNAME` with your actual GitHub username!

**If it asks for password:**
- Use a Personal Access Token (not your GitHub password)
- Get token: https://github.com/settings/tokens
- Select: `repo` scope
- Copy token and use as password

---

### **Step 4: Enable GitHub Pages**

1. Go to your repository on GitHub
2. Click **"Settings"** tab (⚙️)
3. Click **"Pages"** in left sidebar
4. Under **"Build and deployment":**
   - **Source:** Select "GitHub Actions"
5. That's it! No need to click Save

---

### **Step 5: Wait for Deployment** ⏳

1. Click **"Actions"** tab in your repository
2. You'll see "Deploy to GitHub Pages" workflow running
3. Wait 2-3 minutes for it to complete ✅
4. Green checkmark = SUCCESS!

## ❓ **Troubleshooting:**

### **Problem: Git push asks for password**
**Solution:** Use Personal Access Token
1. https://github.com/settings/tokens
2. Generate new token (classic)
3. Check `repo` scope
4. Copy token
5. Use as password

---

### **Problem: 404 Not Found**
**Solution:**
- Wait 5 minutes for deployment
- Check Actions tab - workflow must complete
- Verify GitHub Pages is enabled
- URL must be exact (case-sensitive)

---

### **Problem: Images not showing**
**Solution:**
- Images must be in `wwwroot/images/`
- Re-push: `git add . && git commit -m "fix" && git push`

---

### **Problem: Workflow failed**
**Solution:**
- Click on failed workflow
- Check error message
- Usually: missing .NET configuration (already included!)
